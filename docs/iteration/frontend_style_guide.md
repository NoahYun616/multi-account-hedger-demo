# 前端风格草案

状态：待确认

更新时间：2026-06-04

## 设计定位

合约保险对冲工具不是营销页，也不是普通后台表单。它是一个高风险交易控制台，核心任务是让用户快速判断：

1. 当前是否真实下单。
2. 对冲是否完整。
3. 风险是否失控。
4. 哪些动作需要立刻处理。
5. 当前 Websea 账号保险节点状态是否正常。

因此前端风格应定位为：

> 冷静、专业、信息密度高、风险状态极清晰的交易风控工作台。

不做夸张视觉，不做装饰性大卡片，不做营销式文案，不做过多渐变和动效。

## 设计原则

- 风险优先：危险状态必须比普通数据更醒目。
- 操作克制：真实下单、删除、紧急平仓、切换杠杆等动作必须有明确层级和确认。
- 信息密度高：适合长期盯盘和反复操作，减少大面积留白。
- 状态可扫读：用户一眼能看出正常、预警、危险、暂停、模拟、实盘。
- 数据口径稳定：同类指标在不同页面使用一致命名、颜色和格式。
- 不承诺收益：界面只展示事实、状态、已知结果，不做未来收益预测。

## 整体视觉方向

建议使用浅色专业工作台风格。

原因：

- 当前工具信息较多，浅色底更利于表格、配置项和长时间阅读。
- 风险色在浅色背景上更容易突出。
- 与 Websea 交易所前台不必完全一致，工具应更像风控后台。

可选未来增强：提供深色模式，但第一阶段先固定浅色风格，减少实现复杂度。

## 颜色规范

基础色：

- 页面背景：`#F6F8FA`
- 主内容背景：`#FFFFFF`
- 分隔线：`#E5E7EB`
- 主文本：`#111827`
- 次级文本：`#6B7280`
- 弱提示文本：`#9CA3AF`

品牌/信息色：

- 主操作蓝：`#2563EB`
- 信息蓝底：`#EFF6FF`
- 信息蓝字：`#1D4ED8`

状态色：

- 安全/正常：`#16A34A`
- 安全浅底：`#ECFDF3`
- 预警：`#D97706`
- 预警浅底：`#FFF7ED`
- 危险：`#DC2626`
- 危险浅底：`#FEF2F2`
- 暂停/禁用：`#6B7280`
- 模拟运行：`#64748B`
- 真实下单：`#DC2626`

使用规则：

- 红色只用于真实危险、真实下单、失败、阻断、删除和紧急动作。
- 黄色只用于可恢复但需要关注的风险。
- 绿色只用于检查通过、模拟安全状态、同步完成等正向状态。
- 蓝色用于普通信息、引导和主按钮。

## 字体与排版

字体：

- 中文优先使用系统字体。
- 数字使用等宽或表格式数字效果，便于对齐比较。

字号建议：

- 页面标题：20px / 24px
- 区块标题：16px / 20px
- 正文与表格：14px / 20px
- 辅助说明：12px / 16px
- 状态标签：12px / 16px

排版规则：

- 不使用超大标题。
- 不使用负字距。
- 表格数字右对齐，名称和状态左对齐。
- 关键数值统一保留合理小数位，USDT 建议 2 位，比例建议 2 位。

## 页面框架

建议整体结构为：

1. 顶部状态栏。
2. 左侧导航。
3. 主工作区。
4. 页面内风险提示区。
5. 页面内表格或配置区。

### 顶部状态栏

顶部状态栏应始终展示：

- 当前运行模式：模拟运行 / 真实下单。
- 引擎状态：运行中 / 未运行 / 异常。
- 配置体检状态：通过 / 警告 / 阻断。
- 最近更新时间。
- 当前最高风险等级。

真实下单状态必须常驻红色醒目展示，不允许只出现在配置页。

### 左侧导航

导航应按用户任务分组，而不是按配置文件分组。

建议第一阶段导航：

- 总览
- 对冲监控
- 账号
- 策略
- 交易对
- 配置体检
- 原始配置
- 使用说明

后续接入保险节点后：

- 保险节点
- 策略已知结果

### 主工作区

每个页面顶部应有一行简短说明，但不要写教学长文。

页面主体优先使用：

- 状态条。
- 指标组。
- 数据表格。
- 操作区。
- 折叠详情。

避免：

- 卡片套卡片。
- 大面积装饰。
- 多色块无意义堆叠。
- 把危险操作和普通保存按钮放在同一视觉层级。

## 状态体系

策略单元建议使用统一状态：

- `NORMAL`：正常。
- `WARNING`：预警。
- `PAUSED`：暂停。
- `PROTECTING`：保护处理中。
- `NEED_CONFIRMATION`：等待人工确认。
- `RECOVERED`：已恢复。
- `FAILED`：失败。

状态展示规则：

- 状态标签固定使用同一颜色。
- 状态旁必须给出原因，例如“Websea 可用余额不足”“仓位偏差 12.4% 持续 8 秒”。
- 不要只显示红色失败，不说明失败原因。

## 关键页面风格

### 总览页

目标：让用户 5 秒内知道工具是否安全。

建议模块：

- 运行模式和引擎状态。
- 策略单元风险摘要。
- 当前对冲完成率。
- 当前未对冲敞口。
- API 健康状态。
- 最近告警。

顶部指标建议：

- 已启用策略数。
- 正常策略数。
- 预警策略数。
- 阻断策略数。
- 当前最大未对冲敞口。

### 对冲监控页

目标：看清 Gate 与 Websea 仓位是否匹配。

核心展示：

- Source 当前仓位。
- Hedge 目标仓位。
- Hedge 实际仓位。
- 对冲完成率。
- 偏差数量。
- 偏差比例。
- 未对冲敞口。

表格建议按策略单元 + 交易对展示，每行一个交易对。

### 账号页

目标：管理 API 和账号状态。

规则：

- 密钥永远脱敏。
- 空密钥、失效密钥、权限不足要直接显示状态。
- 新增/编辑账号表单保持紧凑。
- 删除账号必须二次确认。

### 策略页

目标：配置 source / hedge 关系。

规则：

- Source 和 Hedge 左右对照展示。
- `same` / `opposite` 使用分段控件或清晰选择器。
- 杠杆、保证金模式、ratio、followers 分组展示。
- 变更后提示是否需要重启或热加载。

### 交易对页

目标：确保张数、面值、比例不出错。

必须同时展示：

- Gate 张数。
- Gate 合约面值。
- Gate 实际币数量。
- Gate 名义价值。
- Websea 张数。
- Websea 合约面值。
- Websea 实际币数量。
- Websea 名义价值。
- 当前换算比例。

### 配置体检页

目标：真实下单前的阻断检查。

检查结果分三类：

- 通过：绿色。
- 警告：黄色。
- 阻断：红色。

阻断项未解决前，不允许切换真实下单模式。

### 保险节点页

目标：展示当前 Websea 账号保险节点状态。

只展示当前账号维度，不展示平台保险池健康度。

建议展示：

- 是否开启保险。
- 当前账号使用的保险池类型。
- 保险账户余额。
- 已生成节点。
- 有效节点。
- 排队节点。
- 冻结节点。
- 失效节点。
- 风控节点。
- 已领取空投。
- 未领取空投。
- 最近一次节点生成、冻结、失效或风控原因。

## 操作按钮规范

主按钮：

- 保存配置。
- 新增策略。
- 新增交易对。
- 运行体检。

危险按钮：

- 切换真实下单。
- 删除账号。
- 删除策略。
- 紧急暂停。
- 紧急平仓。

危险按钮规则：

- 必须红色。
- 必须二次确认。
- 必须说明后果。
- 默认不放在页面主视觉中心，避免误触。

## 表格规范

表格应尽量稳定、紧凑、可扫读。

规则：

- 状态列靠前。
- 操作列靠右。
- 数字右对齐。
- 交易对、账号、策略名固定宽度。
- 错误原因单独列展示，不塞进状态标签。
- 长错误可折叠查看。

## 告警规范

告警分级：

- 信息：不影响操作。
- 预警：需要关注，但不阻断。
- 阻断：必须处理，禁止继续真实下单或开新仓。
- 紧急：已经发生单边暴露、强平风险或接口异常。

告警展示：

- 顶部状态栏展示最高风险等级。
- 总览页展示最近告警列表。
- 相关页面在对应策略/交易对行内展示告警原因。

## Streamlit 实现建议

当前工具使用 Streamlit，第一阶段不必重构框架，可以通过以下方式统一风格：

- 在 `dashboard.py` 中集中注入 CSS。
- 抽取统一状态标签函数。
- 抽取统一指标块函数。
- 抽取统一告警块函数。
- 抽取统一页面标题函数。
- 表格统一通过格式化函数输出。

建议后续新增：

- `ui_style.py`：颜色、文案、状态标签、CSS。
- `ui_components.py`：指标块、告警条、状态条、确认区。
