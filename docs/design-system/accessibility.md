# Accessibility

## 基础目标

页面必须可读、可操作、可理解。交易风控工具中，可访问性也是安全性的一部分。

## 对比度

要求：

- 正文文本与背景对比度不低于 4.5:1。
- 大号标题不低于 3:1。
- 状态标签不能只依赖颜色。

状态应同时使用：

- 文本。
- 色彩。
- 图标或标签。

## 键盘操作

必须支持：

- Tab 顺序合理。
- Focus 可见。
- Modal 可通过 Esc 或关闭按钮退出。
- 表单可用键盘提交。

## 语义

要求：

- 页面标题使用语义标题层级。
- 表格使用表头。
- 表单控件有标签。
- 图标按钮有说明或 tooltip。

## 错误可理解

错误信息必须：

- 放在错误字段附近或页面顶部总结。
- 用自然语言描述。
- 给出下一步。

## 动效

动效应克制：

- 不使用闪烁。
- 不使用大幅位移。
- Loading 不应造成布局跳动。

## Accessibility 检查项

- 颜色是否不是唯一状态表达。
- Focus 是否可见。
- 表单是否有标签。
- 图标按钮是否有说明。
- 错误是否可理解。
