# Form

## 使用原则

表单用于创建、编辑、配置。交易风控工具中的表单必须优先保证可理解、可校验、可回滚。

## 标签位置规范

默认：

- 中后台配置页使用顶部标签。
- 宽表单或高级配置可使用左侧标签。
- 弹窗表单使用顶部标签，减少横向占用。

标签命名：

- 使用业务语言，不直接暴露程序字段名，除非该字段是配置文件原始字段。
- 原始配置页可以保留 `dry_run`、`state_file` 等字段名。

## 校验规范

校验时机：

- 必填项：提交前校验。
- 数字格式：输入后或提交前校验。
- 高风险配置：提交前校验，并在保存前确认。

校验内容：

- 必填。
- 数字格式。
- 最小值 / 最大值。
- URL 格式。
- 密钥是否为空。
- 交易对映射是否重复。

## 错误提示规范

错误提示必须说明：

- 哪个字段错了。
- 为什么错了。
- 如何修改。

示例：

- “Hedge 比例必须是有限数字。”
- “启用策略账号缺少 API Secret，请在账号页补全。”

禁止：

- 只显示异常堆栈。
- 用英文技术错误直接面向普通用户。

## 必填项规范

必填项需要：

- 明确标记。
- 在提交时集中提示。
- 不因未填而静默失败。

敏感字段：

- 密钥字段默认脱敏。
- 密钥留空代表保留原值时，必须在说明里写清楚。

## 多步骤表单规范

适用于：

- 新增完整策略单元。
- 批量导入账号。
- 真实下单前配置确认。

步骤建议：

1. 基础信息。
2. 账号与权限。
3. 交易对与比例。
4. 风控检查。
5. 确认保存。

每一步必须能返回修改。

## Form 检查项

- 标签是否使用业务语言。
- 必填项是否明确。
- 错误信息是否可行动。
- 密钥是否不会明文展示。
- 高风险保存是否有确认。
