# Feedback

## Toast

Toast 用于轻量结果反馈。

适用：

- 保存成功。
- 删除成功。
- 配置已更新。
- 复制成功。

规则：

- 成功 Toast 自动消失。
- 错误 Toast 应说明原因。
- 高风险操作不只用 Toast，必须配合确认。

## Modal

Modal 用于阻断式确认或复杂表单。

适用：

- 删除账号。
- 删除策略单元。
- 切换真实下单。
- 高风险批量操作。

规则：

- 标题明确动作。
- 正文说明影响。
- 主按钮文案使用动词。
- 危险主按钮使用危险色。

## Alert

Alert 用于页面内持续可见的状态。

适用：

- 真实下单模式已开启。
- 配置存在阻断。
- Websea 资金不足。
- 同步引擎未运行。

规则：

- Alert 应靠近相关内容。
- 真实下单 Alert 必须醒目。
- 不用 Alert 展示普通说明。

## Empty

Empty 用于无数据状态。

必须包含：

- 当前无什么。
- 是否正常。
- 下一步动作。

示例：

- “当前没有交易对。请在交易对页面新增 Gate 与 Websea 的映射。”

## Skeleton

Skeleton 用于页面或卡片加载。

适用：

- 首屏 KPI 加载。
- 表格首次加载。
- 图表加载。

不适用：

- 按钮提交。
- 小范围字段更新。

## Feedback 检查项

- 成功反馈是否轻量。
- 危险操作是否 Modal 确认。
- 真实下单是否 Alert 常驻。
- 空状态是否有下一步。
- Loading 是否不遮挡无关内容。
