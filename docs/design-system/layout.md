# Layout

## 参考来源

本规范参考 Ant Design、Ant Design Pro、Pro Components 的中后台设计方法，并结合 Websea 合约保险对冲工具的交易风控场景沉淀。页面应优先采用稳定、清晰、低噪音的中后台布局，不使用营销页式视觉。

## 页面布局结构

默认页面结构：

1. `Sidebar`：左侧主导航，承载页面入口和基础设置。
2. `Header / PageHeader`：页面标题、简短说明、必要状态和页面级操作。
3. `Content`：核心数据、表格、表单、配置区。
4. `Feedback Layer`：Toast、Modal、Alert、Drawer 等反馈层。

禁止：

- 页面主体先出现大幅宣传内容。
- 卡片套卡片造成层级混乱。
- 同一状态在 Sidebar、Header、KPI 卡片和正文中重复出现。

## Header 规范

Header 只展示页面级信息：

- 页面标题。
- 一句话说明。
- 2 到 3 个关键状态标签。
- 页面级主操作，最多 2 个。

总览页 Header：

- 只保留最近更新时间、全局异常等跨页面状态。
- 运行模式、配置检查、运行健康应放入 KPI 卡片，不在 Header 重复。

其他页面 Header：

- 可展示运行模式、配置检查、最近更新时间。
- 真实下单必须以危险色展示。

## Sidebar 规范

Sidebar 只用于导航和基础设置：

- 顶部展示产品名。
- 语言选择保留在导航上方。
- 菜单按用户任务组织，而不是按配置文件组织。
- 不展示当前状态、启用账号数、策略单元数等主内容重复信息。

默认菜单：

- 总览
- 对冲监控
- 账号
- 策略
- 交易对
- 配置体检
- 原始配置
- 使用说明

后续新增：

- 保险节点
- 告警记录
- 策略结果

## Content 规范

Content 使用浅灰背景 + 白色卡片：

- 页面背景：`color.bg.layout`
- 内容卡片：`color.bg.container`
- 卡片圆角：`radius.lg`
- 卡片间距：`space.24`

内容顺序：

1. 页面核心指标。
2. 当前风险或待处理事项。
3. 主要数据表格。
4. 次级配置和折叠详情。

## PageContainer 规范

参考 Pro Components 的 PageContainer 思路，所有页面应具备统一容器语义：

- `title`：页面名称，短且明确。
- `description`：一句话说明页面用途。
- `extra`：页面级操作，不超过 2 个。
- `tabs`：同一对象的多视图切换才使用。
- `breadcrumb`：仅当页面层级超过 2 层时使用。

当前 Streamlit 实现中，`render_app_header` 承担 PageContainer 的标题区职责。

## 响应式规范

断点：

- `< 640px`：单列布局。
- `640px - 1024px`：双列 KPI，表格横向滚动。
- `> 1024px`：四列 KPI，主表格完整展示。

规则：

- KPI 卡片不得因内容长短改变高度导致布局跳动。
- 表格在窄屏允许横向滚动，不压缩到不可读。
- Sidebar 可收起，但重新展开按钮必须可见且可点击。
- Header 状态标签在窄屏换行，不遮挡标题。

## Layout 检查项

- 页面是否有清晰的 Sidebar / Header / Content 分区。
- Header 是否没有重复 KPI 卡片中的信息。
- Sidebar 是否只承载导航和基础设置。
- 主要任务是否出现在首屏。
- 真实下单状态是否在所有页面可见。
