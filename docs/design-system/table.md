# Table

## 使用原则

表格用于比较、扫描、排序、筛选、分页和批量处理结构化数据。账号列表、策略单元管理列表、交易对列表、告警日志、运行日志、保险节点列表应优先使用表格。

避免把大量结构化数据拆成多个卡片。

表格不是所有结构化信息的唯一默认形态。若页面目标是帮助用户理解少量对象之间的关系、层级、流向或配对状态，应优先考虑关系视图、分组卡片或详情布局。

示例：

- 策略单元的管理、筛选、批量启用：使用表格。
- 总览页中查看 Gate 与 Websea 的 master/sub 配对关系：可使用关系卡片。
- 交易对的大量规则编辑：使用表格。
- 单个交易对的风险解释或转换公式说明：可使用详情卡片。

## 列宽规则

默认列宽：

- 状态列：80-120px。
- 名称列：160-220px。
- 数值列：120-160px。
- 时间列：160-180px。
- 操作列：120-180px。

规则：

- 账号、策略单元、交易对等关键标识不得过窄。
- 长文本默认省略，悬浮或详情展开查看全文。
- 表格最小宽度超过容器时横向滚动，不强行挤压。

## 对齐规则

- 文本左对齐。
- 状态左对齐。
- 数字右对齐。
- 金额右对齐。
- 时间左对齐或等宽左对齐。
- 操作列右对齐。

数字必须使用表格式数字效果，便于比较。

## 排序规则

默认排序：

1. 风险等级高的在前。
2. 异常状态在前。
3. 最近更新时间新的在前。
4. 名称字母或配置顺序。

可排序列：

- 时间。
- 金额。
- 仓位数量。
- 偏差比例。
- 风险等级。

不建议排序：

- 操作列。
- 长文本说明列。
- 密钥脱敏列。

## 筛选规则

常用筛选：

- 状态。
- 交易所。
- 策略单元。
- 交易对。
- 是否启用。
- 风险等级。

筛选控件位置：

- 简单筛选放表格上方。
- 复杂筛选放可折叠筛选区。
- 不在表格每列都塞筛选控件。

## 批量操作规则

仅在确实需要对多行执行同一动作时提供批量操作。

允许：

- 批量启用/禁用 follower。
- 批量导出。
- 批量标记已处理告警。

谨慎：

- 批量删除。
- 批量修改杠杆。
- 批量真实下单相关操作。

高风险批量操作必须二次确认。

## 分页规则

默认：

- 10、20、50 条每页。
- 超过 50 条必须分页或虚拟滚动。
- 分页位于表格右下角。

运行日志、告警日志可使用“加载更多”。

## 操作列规则

操作列只放当前行动作：

- 查看。
- 编辑。
- 启用/禁用。
- 删除。

超过 3 个操作时折叠到“更多”。

危险操作：

- 使用红色或危险样式。
- 放在普通操作之后。
- 必须确认后执行。

## Table 检查项

- 列是否有明确优先级。
- 数字是否右对齐。
- 操作列是否没有塞太多按钮。
- 危险操作是否二次确认。
- 空表格是否有下一步提示。
